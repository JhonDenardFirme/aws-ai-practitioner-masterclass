Module Introduction

Every module in this masterclass — Foundation Model selection, optimization with RAG and fine-tuning, security and governance, prompt engineering, and hands-on building with PartyRock — rests on a shared foundation of terminology and concepts. This module establishes that foundation: what AI, ML, DL, and Generative AI actually mean relative to each other, how models are trained and architected, and which AWS services make up the AI stack everything else in this course builds on.

The module moves from the AI/ML/DL/Generative AI hierarchy, through the ML pipeline and neural network fundamentals, into Foundation Models and the specific architectures — LLMs, diffusion models, GANs, VAEs — that generative systems use, before closing with the AWS AI stack itself and the cost trade-offs involved in deploying it.

A The Artificial Intelligence Ecosystem A
A1AI, ML, DL, and Generative AI Defined

Four terms get used almost interchangeably in casual conversation, but each names a progressively narrower field.

Artificial Intelligence (AI)
  • Machines simulating human intelligence — the broadest umbrella term
Machine Learning (ML)
  • Systems that learn patterns from data rather than following fixed rules
Deep Learning (DL)
  • Neural networks with a layered structure, a specific approach within ML
Generative AI
  • Produces new data or content, rather than only analyzing existing data
Key Points
  • Generative AI creates — traditional ML analyzes. The module's core distinction.
A2The Hierarchy

These four fields nest strictly inside one another: AI is the umbrella field, ML is a subset of AI, DL is a subset of ML, and Generative AI is a subset of DL. Every generative AI system is a deep learning system, every deep learning system is a machine learning system, and every machine learning system is an AI system — but the reverse isn't true.

Key Points
  • Hierarchy: AI → ML → DL → Generative AI, each one nested inside the last
B Machine Learning Fundamentals B
B1The ML Pipeline

Every ML system moves through five stages: Data Collection (gather raw data), Data Preparation (clean and preprocess), Training (learn patterns from data), Evaluation (measure performance), and Iteration (improve the model based on what evaluation reveals).

Model quality equals data quality — the classic “garbage in, garbage out” principle. A model is only ever as good as the data used to train it, regardless of how sophisticated the algorithm is.

Key Points
  • ML Pipeline: Data Collection → Data Preparation → Training → Evaluation → Iteration
  • Model quality = data quality — bad data guarantees a bad model
B2Types of Data

Data feeding an ML pipeline splits along two independent axes: whether it's labeled, and whether it's structured.

Labeled Data
  • Data paired with correct descriptions or answers
Unlabeled Data
  • Data with no attached labels or answers
Structured Data
  • Organized into tables/databases — tabular (rows/columns) or time-series (sequential)
Unstructured Data
  • No predefined format — text (documents, posts) or image (photos, video)
Key Points
  • Four data types: Labeled, Unlabeled, Structured (tabular/time-series), Unstructured (text/image)
B3Learning Types

How a model learns from data falls into four paradigms, distinguished mainly by how much labeled data is available and whether a reward signal exists.

Supervised Learning
  • Learns from labeled data to predict outcomes for unseen data
Unsupervised Learning
  • Finds hidden patterns in unlabeled data with no target answer
Reinforcement Learning
  • Learns through a reward-based feedback signal, not fixed labels
Semi-Supervised Learning
  • Uses a mix of partial labeled data alongside unlabeled data
Key Points
  • Four learning types: Supervised, Unsupervised, Reinforcement, Semi-supervised
B4Inferencing

Inference is making predictions using an already-trained model. It comes in two forms: Batch Inference processes large datasets all at once and takes time, while Real-Time Inference returns instant predictions for a single request.

Key Points
  • Batch Inference = large datasets, takes time. Real-Time Inference = instant, single request.
C Deep Learning Fundamentals C
C1Neural Network Structure

Deep Learning uses brain-inspired artificial neural networks. Every network is built from nodes connected by weighted relationships, organized into three layer types.

Input Layer
  • Receives the raw data entering the network
Hidden Layer
  • Processes patterns between input and output — a network can have many of these
Output Layer
  • Produces the network's final result
Key Points
  • Neural networks are built from nodes (neurons), weighted connections, and three layer types: Input, Hidden, Output
C2The Learning Process

A neural network learns by adjusting the weights on its connections, detecting patterns in the training data, and generalizing what it learns to new, unseen data. Nodes effectively “talk to each other” through these weighted connections, incrementally improving predictions as weights are tuned.

Key Points
  • Learning process: adjust weights → detect patterns → generalize to new data
C3Applications of Neural Networks

Two domains dominate practical neural network applications. Computer Vision handles image and video understanding — image classification, object detection, segmentation. Natural Language Processing (NLP) handles language understanding — text classification, sentiment analysis, machine translation, text generation.

Key Points
  • Computer Vision: image classification, object detection, segmentation
  • NLP: text classification, sentiment analysis, machine translation, text generation
D Generative AI Fundamentals D
D1What Generative AI Is

Generative AI creates text, images, audio, and code — its key driver is the combination of massive compute power and large-scale datasets that became available in the last several years.

Key Points
  • Generative AI's key driver: compute power + large datasets, working together
D2Foundation Models (FMs)

Foundation Models are pretrained on massive data, with multi-task capability — one model can perform many different tasks — and general-purpose adaptability across domains. [+] Because they're pretrained at internet scale, FMs make an excellent starting point for a new project: development is faster since the hardest, most expensive part — initial training — is already done.

Key Points
  • FMs are pretrained, multi-task, and general-purpose — adaptable across domains
D3The FM Lifecycle

Every Foundation Model moves through six stages from raw data to a continuously improving production system.

Data Selection
  • Gathering large unlabeled datasets
Pre-Training
  • Self-supervised learning; the model's own structure auto-generates training labels. Continuous Pre-Training later expands the model's knowledge across domains
Optimization
  • Improving performance beyond the raw pre-trained state
Evaluation
  • Measuring effectiveness using metrics and benchmarks
Deployment
  • Integrating the model into production systems
Feedback Loop
  • Continuous improvement based on real-world usage
Key Points
  • FMs use self-supervised learning — no labels required, the module's key insight
  • [+] Amazon Bedrock provides FMs from leading companies including Anthropic, Meta, Mistral AI, and Amazon itself
E Generative Model Architectures E
E1Large Language Models (LLMs)

LLMs generate human-like text, built on the Transformer architecture — the core design behind essentially every state-of-the-art text model, trained on vast data spanning the internet, books, and open sources. Three concepts explain how an LLM actually processes text.

Tokens
  • The smallest units of text — words, phrases, spaces, or characters — providing standardized input
Embeddings
  • Numerical representations of words, positioned so related concepts sit closer together
Vectors
  • The underlying numerical form that captures semantic relationships between tokens
Key Points
  • LLMs use tokens, embeddings, and vectors together to understand and generate text
  • Related words (cat, feline, kitten) end up as vectors positioned close together
E2Diffusion Models

Diffusion is a deep learning architecture that starts with pure noise or random data, then adds meaningful information to clear that noise and build understanding. It has two directions: Forward Diffusion adds noise to clear data, used during training; Reverse Diffusion removes noise until data is generated, used during inference.

Key Points
  • Forward Diffusion: adds noise, used for training. Reverse Diffusion: removes noise, used for inference.
E3Multimodal Models

Multimodal Models process multiple data types together — text plus image (captioned images), or image plus audio. They're used for advanced tasks: answering complex questions, automating video captioning, and creating graphics directly from text instructions.

Key Points
  • Multimodal examples: Text + Image, Image + Audio
  • Advanced use cases: complex Q&A, video captioning, text-to-graphics generation
E4Generative Adversarial Networks (GANs)

A GAN pits two neural networks against each other: a Generator creates synthetic/fake data, and a Discriminator tries to tell real data from the generator's fake data. The Generator's goal is to fool the Discriminator; the Discriminator's goal is to correctly identify real versus generated data — training continues until the generated data becomes indistinguishable from real data.

Key Points
  • GAN = Generator (creates fakes) vs Discriminator (detects fakes), trained adversarially until fakes are indistinguishable from real data
E5Variational Autoencoders (VAEs)

A VAE combines ideas from autoencoders and variational inference (a Bayesian statistics technique). An Encoder compresses input data into a lower-dimensional space; a Decoder takes that low-dimensional representation and reconstructs it back into data. That lower-dimensional space is constrained to follow a specific probability distribution — usually Gaussian — which is what makes the generated results predictable rather than arbitrary.

Key Points
  • VAE = Encoder (compresses) + Decoder (reconstructs), with a Gaussian-distributed latent space producing predictable results
F Model Optimization Techniques F
F1Prompt Engineering

Prompt Engineering designs better inputs to a Foundation Model, combining four components into a complete prompt.

Instruction
  • Defines the task the model should perform
Context
  • Supporting information that guides the model's response
Input Data
  • The raw content being processed
Output Indicator
  • Specifies the desired output format
Key Points
  • Prompt components: Instruction, Context, Input Data, Output Indicator — the same four-part structure covered in depth in Module 8
F2Fine-Tuning

Fine-Tuning retrains a Foundation Model with a smaller, narrower, more specific dataset using supervised learning, improving performance for a targeted use case. Two named approaches: Instruction Tuning uses examples of how the model should respond (a form of prompt tuning), and RLHF (Reinforcement Learning from Human Feedback) uses human ratings and comments to shape behavior.

Key Points
  • Fine-tuning uses supervised learning on a narrower dataset
  • Two approaches: Instruction Tuning (response examples), RLHF (human feedback)
F3Retrieval-Augmented Generation (RAG)

RAG supplies relevant domain data — retrieved from documents and files — to produce better, more grounded responses. RAG does NOT change the model's weights, unlike fine-tuning — it changes what the model sees at inference time, not what the model has learned.

Key Points
  • RAG ≠ Fine-Tuning: RAG never changes model weights — the module's key insight
G AWS AI/ML Infrastructure G
G1The ML Framework Layer: Amazon SageMaker

Amazon SageMaker is the foundation-layer service for building, training, and deploying custom ML models — the layer developers use when they need full control over the model lifecycle rather than a pre-built API.

Key Points
  • SageMaker covers the full custom model lifecycle: build, train, deploy
G2The AI/ML Services Layer

Ten pre-built AWS services sit above SageMaker, each solving one specific AI task without requiring custom model development.

Amazon Comprehend
  • NLP insights extraction
Amazon Translate
  • Neural machine translation
Amazon Textract
  • Document data extraction, OCR
Amazon Lex
  • Chatbot/voice interfaces — ASR + NLU, built for AI agents
Amazon Polly
  • Text-to-speech, voiceover generation
Amazon Transcribe
  • Speech-to-text (Automatic Speech Recognition)
Amazon Rekognition
  • Image and video analysis
Amazon Kendra
  • Intelligent enterprise search
Amazon Personalize
  • Recommendation generation
AWS DeepRacer
  • Reinforcement learning platform
Key Points
  • Ten AI/ML services, each targeting one specific task: NLP, translation, OCR, voice, speech, vision, search, recommendations, RL
G3The Generative AI Layer

Four services form the top, generative-AI-specific layer of the AWS AI stack.

SageMaker JumpStart
  • Prebuilt ML solutions ready to deploy
Amazon Bedrock
  • Managed access to multiple Foundation Models
Amazon Q
  • AI assistant for business use
Amazon Q Developer
  • AI code generation assistant
Key Points
  • Generative AI layer: SageMaker JumpStart, Amazon Bedrock, Amazon Q, Amazon Q Developer
H Advantages of AWS AI Solutions H
H1Six Core Advantages

AWS's AI advantage rests on six pillars, together enabling rapid AI deployment without heavy infrastructure investment.

Faster Development
  • Prebuilt tools and APIs shortcut months of infrastructure work
Scalability
  • Global infrastructure scales with demand automatically
Cost Efficiency
  • Pay-as-you-go pricing avoids large upfront investment
Flexibility
  • Wide access to multiple model families and providers
Integration
  • Seamless connection across the broader AWS ecosystem
Security
  • Enterprise-grade protection built into the platform
Key Points
  • AWS enables rapid AI deployment without heavy infrastructure — the module's key insight for this section
I Cost Considerations in AWS AI I
I1Cost Trade-off Factors

Seven factors determine an AWS AI deployment's actual cost, and most of them trade directly against performance or reliability.

Responsiveness
  • Latency versus cost — faster response generally costs more
Availability
  • Uptime versus resource cost — higher availability requires more redundant capacity
Redundancy
  • Backup systems increase cost directly
Performance
  • Compute power versus pricing — more capable instances cost more
Token-Based Pricing
  • Cost scales directly with usage volume
Provisioned Throughput
  • Reserved capacity costs more but guarantees availability
Custom Models
  • Training a custom model costs more than using a pre-trained one
Key Points
  • Balance cost, performance, and scalability — the module's key insight for this section
Module Summary
  • AI, ML, DL, and Generative AI form a strict hierarchy — each one nested inside the last — and Generative AI creates content while traditional ML only analyzes it.
  • The ML pipeline (Data Collection → Preparation → Training → Evaluation → Iteration) depends entirely on data quality; Foundation Models specifically use self-supervised learning, requiring no labels.
  • Generative architectures solve the same problem differently: LLMs use tokens/embeddings/vectors, Diffusion Models use forward/reverse noise, GANs pit a Generator against a Discriminator, and VAEs compress and reconstruct through a Gaussian-distributed latent space.
  • Prompt Engineering, Fine-Tuning, and RAG are the three model optimization techniques — critically, RAG never changes model weights, unlike Fine-Tuning.
  • AWS's AI stack layers Amazon SageMaker (custom models), ten AI/ML services (Comprehend, Translate, Textract, and more), and a Generative AI layer (Bedrock, Amazon Q) — delivering faster development and scalability, balanced against real cost trade-offs in responsiveness, availability, and throughput.
