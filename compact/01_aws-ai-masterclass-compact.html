A The Artificial Intelligence Ecosystem A
A1AI, ML, DL, and Generative AI Defined

Four terms get used almost interchangeably in casual conversation, but each names a progressively narrower field.

Artificial Intelligence (AI)
  • Machines simulating human intelligence — the broadest umbrella term
Machine Learning (ML)
  • Systems that learn patterns from data rather than following fixed rules
Deep Learning (DL)
  • Neural networks with a layered structure, a specific approach within ML
Generative AI
  • Produces new data or content, rather than only analyzing existing data
KeyGenerative AI creates — traditional ML analyzes. The module's core distinction.
A2The Hierarchy

These four fields nest strictly inside one another: AI is the umbrella field, ML is a subset of AI, DL is a subset of ML, and Generative AI is a subset of DL.

KeyHierarchy: AI → ML → DL → Generative AI, each one nested inside the last
B Machine Learning Fundamentals B
B1The ML Pipeline

Every ML system moves through five stages: Data Collection (gather raw data), Data Preparation (clean and preprocess), Training (learn patterns from data), Evaluation (measure performance), and Iteration (improve the model based on what evaluation reveals). Model quality equals data quality — the classic “garbage in, garbage out” principle.

KeyML Pipeline: Data Collection → Data Preparation → Training → Evaluation → Iteration›Model quality = data quality — bad data guarantees a bad model
B2Types of Data

Data feeding an ML pipeline splits along two independent axes: whether it's labeled, and whether it's structured.

Labeled Data
  • Data paired with correct descriptions or answers
Unlabeled Data
  • Data with no attached labels or answers
Structured Data
  • Organized into tables/databases — tabular (rows/columns) or time-series (sequential)
Unstructured Data
  • No predefined format — text (documents, posts) or image (photos, video)
KeyFour data types: Labeled, Unlabeled, Structured (tabular/time-series), Unstructured (text/image)
B3Learning Types

How a model learns from data falls into four paradigms, distinguished mainly by how much labeled data is available and whether a reward signal exists.

Supervised Learning
  • Learns from labeled data to predict outcomes for unseen data
Unsupervised Learning
  • Finds hidden patterns in unlabeled data with no target answer
Reinforcement Learning
  • Learns through a reward-based feedback signal, not fixed labels
Semi-Supervised Learning
  • Uses a mix of partial labeled data alongside unlabeled data
KeyFour learning types: Supervised, Unsupervised, Reinforcement, Semi-supervised
B4Inferencing

Inference is making predictions using an already-trained model.

KeyBatch Inference = large datasets, takes time. Real-Time Inference = instant, single request.
C Deep Learning Fundamentals C
C1Neural Network Structure

Deep Learning uses brain-inspired artificial neural networks.

Input Layer
  • Receives the raw data entering the network
Hidden Layer
  • Processes patterns between input and output — a network can have many of these
Output Layer
  • Produces the network's final result
KeyNeural networks are built from nodes (neurons), weighted connections, and three layer types: Input, Hidden, Output
C2The Learning Process

A neural network learns by adjusting the weights on its connections, detecting patterns in the training data, and generalizing what it learns to new, unseen data.

KeyLearning process: adjust weights → detect patterns → generalize to new data
C3Applications of Neural Networks

Two domains dominate practical neural network applications.

KeyComputer Vision: image classification, object detection, segmentation›NLP: text classification, sentiment analysis, machine translation, text generation
D Generative AI Fundamentals D
D1What Generative AI Is

Generative AI creates text, images, audio, and code — its key driver is the combination of massive compute power and large-scale datasets that became available in the last several years.

KeyGenerative AI's key driver: compute power + large datasets, working together
D2Foundation Models (FMs)

Foundation Models are pretrained on massive data, with multi-task capability — one model can perform many different tasks — and general-purpose adaptability across domains.

KeyFMs are pretrained, multi-task, and general-purpose — adaptable across domains
D3The FM Lifecycle

Every Foundation Model moves through six stages from raw data to a continuously improving production system.

Data Selection
  • Gathering large unlabeled datasets
Pre-Training
  • Self-supervised learning; the model's own structure auto-generates training labels. Continuous Pre-Training later expands the model's knowledge across domains
Optimization
  • Improving performance beyond the raw pre-trained state
Evaluation
  • Measuring effectiveness using metrics and benchmarks
Deployment
  • Integrating the model into production systems
Feedback Loop
  • Continuous improvement based on real-world usage
KeyFMs use self-supervised learning — no labels required, the module's key insight›[+] Amazon Bedrock provides FMs from leading companies including Anthropic, Meta, Mistral AI, and Amazon itself
E Generative Model Architectures E
E1Large Language Models (LLMs)

LLMs generate human-like text, built on the Transformer architecture — the core design behind essentially every state-of-the-art text model, trained on vast data spanning the internet, books, and open sources.

Tokens
  • The smallest units of text — words, phrases, spaces, or characters — providing standardized input
Embeddings
  • Numerical representations of words, positioned so related concepts sit closer together
Vectors
  • The underlying numerical form that captures semantic relationships between tokens
KeyLLMs use tokens, embeddings, and vectors together to understand and generate text›Related words (cat, feline, kitten) end up as vectors positioned close together
E2Diffusion Models

Diffusion is a deep learning architecture that starts with pure noise or random data, then adds meaningful information to clear that noise and build understanding.

KeyForward Diffusion: adds noise, used for training. Reverse Diffusion: removes noise, used for inference.
E3Multimodal Models

Multimodal Models process multiple data types together — text plus image (captioned images), or image plus audio.

KeyMultimodal examples: Text + Image, Image + Audio›Advanced use cases: complex Q&A, video captioning, text-to-graphics generation
E4Generative Adversarial Networks (GANs)

A GAN pits two neural networks against each other: a Generator creates synthetic/fake data, and a Discriminator tries to tell real data from the generator's fake data.

KeyGAN = Generator (creates fakes) vs Discriminator (detects fakes), trained adversarially until fakes are indistinguishable from real data
E5Variational Autoencoders (VAEs)

A VAE combines ideas from autoencoders and variational inference (a Bayesian statistics technique).

KeyVAE = Encoder (compresses) + Decoder (reconstructs), with a Gaussian-distributed latent space producing predictable results
F Model Optimization Techniques F
F1Prompt Engineering

Prompt Engineering designs better inputs to a Foundation Model, combining four components into a complete prompt.

Instruction
  • Defines the task the model should perform
Context
  • Supporting information that guides the model's response
Input Data
  • The raw content being processed
Output Indicator
  • Specifies the desired output format
KeyPrompt components: Instruction, Context, Input Data, Output Indicator — the same four-part structure covered in depth in Module 8
F2Fine-Tuning

Fine-Tuning retrains a Foundation Model with a smaller, narrower, more specific dataset using supervised learning, improving performance for a targeted use case.

KeyFine-tuning uses supervised learning on a narrower dataset›Two approaches: Instruction Tuning (response examples), RLHF (human feedback)
F3Retrieval-Augmented Generation (RAG)

RAG supplies relevant domain data — retrieved from documents and files — to produce better, more grounded responses.

KeyRAG ≠ Fine-Tuning: RAG never changes model weights — the module's key insight

Amazon SageMaker is the foundation-layer service for building, training, and deploying custom ML models — the layer developers use when they need full control over the model lifecycle rather than a pre-built API.

KeySageMaker covers the full custom model lifecycle: build, train, deploy

Four services form the top, generative-AI-specific layer of the AWS AI stack.

I Cost Considerations in AWS AI I
I1Cost Trade-off Factors

Seven factors determine an AWS AI deployment's actual cost, and most of them trade directly against performance or reliability.

Responsiveness
  • Latency versus cost — faster response generally costs more
Availability
  • Uptime versus resource cost — higher availability requires more redundant capacity
Redundancy
  • Backup systems increase cost directly
Performance
  • Compute power versus pricing — more capable instances cost more
Token-Based Pricing
  • Cost scales directly with usage volume
Provisioned Throughput
  • Reserved capacity costs more but guarantees availability
Custom Models
  • Training a custom model costs more than using a pre-trained one
KeyBalance cost, performance, and scalability — the module's key insight for this section
Module Summary
  • AI, ML, DL, and Generative AI form a strict hierarchy — each one nested inside the last — and Generative AI creates content while traditional ML only analyzes it.
  • The ML pipeline (Data Collection → Preparation → Training → Evaluation → Iteration) depends entirely on data quality; Foundation Models specifically use self-supervised learning, requiring no labels.
  • Generative architectures solve the same problem differently: LLMs use tokens/embeddings/vectors, Diffusion Models use forward/reverse noise, GANs pit a Generator against a Discriminator, and VAEs compress and reconstruct through a Gaussian-distributed latent space.
  • Prompt Engineering, Fine-Tuning, and RAG are the three model optimization techniques — critically, RAG never changes model weights, unlike Fine-Tuning.
  • AWS's AI stack layers Amazon SageMaker (custom models), ten AI/ML services (Comprehend, Translate, Textract, and more), and a Generative AI layer (Bedrock, Amazon Q) — delivering faster development and scalability, balanced against real cost trade-offs in responsiveness, availability, and throughput.
