Fundamentals of Machine Learning and Artificial Intelligence
Module 1. Fundamentals of Machine Learning and Artificial Intelligence ├─── A. The Artificial Intelligence Ecosystem │ ├─── A1. AI, ML, DL, and Generative AI Defined │ │ ├─── Artificial Intelligence (AI) — Machines simulating human intelligence — the broadest umbrella term │ │ ├─── Machine Learning (ML) — Systems that learn patterns from data rather than following fixed rules │ │ ├─── Deep Learning (DL) — Neural networks with a layered structure, a specific approach within ML │ │ └─── Generative AI — Produces new data or content, rather than only analyzing existing data │ └─── A2. The Hierarchy │ └─── Hierarchy: AI → ML → DL → Generative AI, each one nested inside the last ├─── B. Machine Learning Fundamentals │ ├─── B1. The ML Pipeline │ │ ├─── ML Pipeline: Data Collection → Data Preparation → Training → Evaluation → Iteration │ │ └─── Model quality = data quality — bad data guarantees a bad model │ ├─── B2. Types of Data │ │ ├─── Labeled Data — Data paired with correct descriptions or answers │ │ ├─── Unlabeled Data — Data with no attached labels or answers │ │ ├─── Structured Data — Organized into tables/databases — tabular (rows/columns) or time-series (sequential) │ │ └─── Unstructured Data — No predefined format — text (documents, posts) or image (photos, video) │ ├─── B3. Learning Types │ │ ├─── Supervised Learning — Learns from labeled data to predict outcomes for unseen data │ │ ├─── Unsupervised Learning — Finds hidden patterns in unlabeled data with no target answer │ │ ├─── Reinforcement Learning — Learns through a reward-based feedback signal, not fixed labels │ │ └─── Semi-Supervised Learning — Uses a mix of partial labeled data alongside unlabeled data │ └─── B4. Inferencing │ └─── Batch Inference = large datasets, takes time. Real-Time Inference = instant, single request. ├─── C. Deep Learning Fundamentals │ ├─── C1. Neural Network Structure │ │ ├─── Input Layer — Receives the raw data entering the network │ │ ├─── Hidden Layer — Processes patterns between input and output — a network can have many of these │ │ └─── Output Layer — Produces the network's final result │ ├─── C2. The Learning Process │ │ └─── Learning process: adjust weights → detect patterns → generalize to new data │ └─── C3. Applications of Neural Networks │ ├─── Computer Vision: image classification, object detection, segmentation │ └─── NLP: text classification, sentiment analysis, machine translation, text generation ├─── D. Generative AI Fundamentals │ ├─── D1. What Generative AI Is │ │ └─── Generative AI's key driver: compute power + large datasets, working together │ ├─── D2. Foundation Models (FMs) │ │ └─── FMs are pretrained, multi-task, and general-purpose — adaptable across domains │ └─── D3. The FM Lifecycle │ ├─── Data Selection — Gathering large unlabeled datasets │ ├─── Pre-Training — Self-supervised learning; the model's own structure auto-generates training labels. Continuous Pre-Training later expands the model's knowledge across domains │ ├─── Optimization — Improving performance beyond the raw pre-trained state │ ├─── Evaluation — Measuring effectiveness using metrics and benchmarks │ ├─── Deployment — Integrating the model into production systems │ └─── Feedback Loop — Continuous improvement based on real-world usage ├─── E. Generative Model Architectures │ ├─── E1. Large Language Models (LLMs) │ │ ├─── Tokens — The smallest units of text — words, phrases, spaces, or characters — providing standardized input │ │ ├─── Embeddings — Numerical representations of words, positioned so related concepts sit closer together │ │ └─── Vectors — The underlying numerical form that captures semantic relationships between tokens │ ├─── E2. Diffusion Models │ │ └─── Forward Diffusion: adds noise, used for training. Reverse Diffusion: removes noise, used for inference. │ ├─── E3. Multimodal Models │ │ ├─── Multimodal examples: Text + Image, Image + Audio │ │ └─── Advanced use cases: complex Q&A, video captioning, text-to-graphics generation │ ├─── E4. Generative Adversarial Networks (GANs) │ │ └─── GAN = Generator (creates fakes) vs Discriminator (detects fakes), trained adversarially until fakes are indistinguishable from real data │ └─── E5. Variational Autoencoders (VAEs) │ └─── VAE = Encoder (compresses) + Decoder (reconstructs), with a Gaussian-distributed latent space producing predictable results ├─── F. Model Optimization Techniques │ ├─── F1. Prompt Engineering │ │ ├─── Instruction — Defines the task the model should perform │ │ ├─── Context — Supporting information that guides the model's response │ │ ├─── Input Data — The raw content being processed │ │ └─── Output Indicator — Specifies the desired output format │ ├─── F2. Fine-Tuning │ │ ├─── Fine-tuning uses supervised learning on a narrower dataset │ │ └─── Two approaches: Instruction Tuning (response examples), RLHF (human feedback) │ └─── F3. Retrieval-Augmented Generation (RAG) │ └─── RAG ≠ Fine-Tuning: RAG never changes model weights — the module's key insight ├─── G. AWS AI/ML Infrastructure │ ├─── G1. The ML Framework Layer: Amazon SageMaker │ │ └─── SageMaker covers the full custom model lifecycle: build, train, deploy │ ├─── G2. The AI/ML Services Layer │ │ ├─── Amazon Comprehend — NLP insights extraction │ │ ├─── Amazon Translate — Neural machine translation │ │ ├─── Amazon Textract — Document data extraction, OCR │ │ ├─── Amazon Lex — Chatbot/voice interfaces — ASR + NLU, built for AI agents │ │ ├─── Amazon Polly — Text-to-speech, voiceover generation │ │ ├─── Amazon Transcribe — Speech-to-text (Automatic Speech Recognition) │ │ ├─── Amazon Rekognition — Image and video analysis │ │ ├─── Amazon Kendra — Intelligent enterprise search │ │ ├─── Amazon Personalize — Recommendation generation │ │ └─── AWS DeepRacer — Reinforcement learning platform │ └─── G3. The Generative AI Layer │ ├─── SageMaker JumpStart — Prebuilt ML solutions ready to deploy │ ├─── Amazon Bedrock — Managed access to multiple Foundation Models │ ├─── Amazon Q — AI assistant for business use │ └─── Amazon Q Developer — AI code generation assistant ├─── H. Advantages of AWS AI Solutions │ └─── H1. Six Core Advantages │ ├─── Faster Development — Prebuilt tools and APIs shortcut months of infrastructure work │ ├─── Scalability — Global infrastructure scales with demand automatically │ ├─── Cost Efficiency — Pay-as-you-go pricing avoids large upfront investment │ ├─── Flexibility — Wide access to multiple model families and providers │ ├─── Integration — Seamless connection across the broader AWS ecosystem │ └─── Security — Enterprise-grade protection built into the platform └─── I. Cost Considerations in AWS AI └─── I1. Cost Trade-off Factors ├─── Responsiveness — Latency versus cost — faster response generally costs more ├─── Availability — Uptime versus resource cost — higher availability requires more redundant capacity ├─── Redundancy — Backup systems increase cost directly ├─── Performance — Compute power versus pricing — more capable instances cost more ├─── Token-Based Pricing — Cost scales directly with usage volume ├─── Provisioned Throughput — Reserved capacity costs more but guarantees availability └─── Custom Models — Training a custom model costs more than using a pre-trained one
