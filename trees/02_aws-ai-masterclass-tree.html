Core AI Applications │ ├─── C1. Computer Vision │ │ ├─── Image Classification — Labels an image with its overall category │ │ ├─── Object Detection — Identifies and locates specific objects within an image │ │ └─── Segmentation — Performs pixel-level analysis, isolating exact shapes │ ├─── C2. Natural Language Processing (NLP) │ │ ├─── Text Classification — Assigns a category label to a piece of text │ │ ├─── Sentiment Analysis — Determines the emotional tone of text │ │ ├─── Translation — Converts text between languages │ │ └─── Text Generation — Produces new, coherent text │ ├─── C3. Intelligent Document Processing (IDP) │ │ ├─── Data Extraction — Pulling structured values out of unstructured documents │ │ ├─── Classification — Sorting documents into the correct category │ │ └─── Summarization — Condensing long documents into key points │ └─── C4. Machine Learning Fundamentals in Practice │ ├─── D1. When ML Is Appropriate │ │ ├─── Complex Rules — The correct logic is too difficult to define manually │ │ ├─── High Variability — Many influencing factors make a fixed rule set brittle │ │ └─── Large Scale — The problem involves massive datasets no manual process could handle │ └─── D2. When ML Is NOT Appropriate │ └─── Do not use ML if a rule-based solution already works — the module's core caution ├─── E. Machine Learning Techniques and Use Cases │ ├─── E1. Supervised Learning: Classification vs. Regression │ │ ├─── Classification = categorical prediction (fraud detection, image classification, customer retention, diagnostics) │ │ └─── Regression = continuous prediction (weather forecasting, market trends, life expectancy, population growth) │ ├─── E2. Unsupervised Learning: Clustering vs. Dimensionality Reduction │ │ ├─── Clustering: customer segmentation, targeted marketing, recommendation systems │ │ └─── Dimensionality Reduction: data visualization, compression, feature extraction │ └─── E3. Reinforcement Learning │ ├─── Agent — The decision-maker taking actions │ ├─── Environment — The space the agent interacts with │ └─── Feedback — Rewards and penalties that shape future decisions ├─── F. Generative AI Overview │ └─── F1. What Generative AI Is, and Its Content Types │ ├─── Text — Conversations, summaries │ ├─── Image — Visual content │ ├─── Video — Generated scenes │ ├─── Audio — Music, speech │ └─── Code — Program generation ├─── G. Capabilities of Generative AI │ └─── G1. Seven Capabilities │ ├─── Adaptability — Flexible across tasks and domains │ ├─── Responsiveness — Generates output in real time │ ├─── Simplicity — Automates otherwise complex processes │ ├─── Creativity — Generates novel ideas and designs │ ├─── Data Efficiency — Learns effectively from limited data │ ├─── Personalization — Tailors outputs to individual users │ └─── Scalability — Produces content at scale ├─── H. Challenges of Generative AI │ ├─── H1. Seven Risks │ │ ├─── Regulatory Violations — Exposure of sensitive or restricted data │ │ ├─── Social Risks — Harmful or reputationally damaging content │ │ ├─── Privacy Concerns — Misuse of personal data │ │ ├─── Toxicity — Offensive or inappropriate outputs │ │ ├─── Hallucinations — Plausible but false generated information │ │ ├─── Interpretability — Difficulty understanding why an output was produced │ │ └─── Nondeterminism — Inconsistent outputs across runs │ └─── H2. Mitigation Strategies │ ├─── Data Anonymization — Protects sensitive data from exposure │ ├─── Model Testing — Evaluates behavior before deployment │ ├─── Security Measures — Encryption and firewalls protecting the system │ ├─── Data Curation — Removes harmful content from training data │ ├─── Guardrails — Filters outputs before they reach the user │ ├─── Verification — Cross-checks outputs against trusted sources │ └─── Monitoring — Detects inconsistencies in ongoing production use ├─── I. Model Selection Factors │ ├─── I1. Core Selection Factors │ │ ├─── Model Type — Task-specific capability match │ │ ├─── Performance — Accuracy and reliability under real conditions │ │ ├─── Capabilities — The range of tasks the model actually supports │ │ ├─── Constraints — Resource and deployment limitations │ │ ├─── Compliance — Ethical and regulatory requirements │ │ └─── Cost — The trade-off between model size and speed │ ├─── I2. Model Examples │ │ ├─── AI21 Jurassic-2 — Text generation, summarization │ │ ├─── Amazon Titan — Embeddings, search, Q&A │ │ ├─── Anthropic Claude — Chat, reasoning, coding │ │ ├─── Stable Diffusion — Image generation │ │ ├─── Cohere Command — NLP tasks │ │ └─── Meta Llama — General text processing │ └─── I3. Constraints and Cost Trade-offs │ ├─── Compute Resources — GPU, CPU, and memory availability │ ├─── Data Availability — Quality and size of usable data │ └─── Deployment Needs — Cloud versus on-premises requirements └─── J. Business Metrics for Generative AI ├─── J1. Five Business Metrics │ ├─── User Satisfaction — Direct feedback on the experience │ ├─── Average Revenue Per User (ARPU) — Revenue generated per user │ ├─── Cross-Domain Performance — Effectiveness across multiple use cases at once │ ├─── Conversion Rate — Rate of desired action completion │ └─── Efficiency — Resource and time optimization achieved └─── J2. Key Insights on Metrics ├─── Metrics are use-case-dependent, not universal └─── Success = technical performance + business value — the module's closing insight
