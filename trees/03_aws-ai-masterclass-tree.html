Module 3. Responsible Artificial Intelligence Practices ├─── A. Responsible AI Fundamentals │ ├─── A1. What Responsible AI Is │ │ ├─── Design — Ethical planning before any code is written │ │ ├─── Development — Active bias mitigation during building │ │ ├─── Deployment — Safe release into production │ │ ├─── Monitoring — Continuous oversight after launch │ │ └─── Evaluation — Ongoing performance and ethics checks │ ├─── A2. Organizational Requirements │ │ ├─── Transparency + Accountability — Monitoring mechanisms that make behavior visible │ │ ├─── Leadership Oversight — Responsible AI governance owned at the top, not delegated away │ │ ├─── Expert Teams — Staff specifically trained in ethical AI practice │ │ └─── Guideline Compliance — Following established standards, not improvising │ ├─── A3. Traditional AI vs. Generative AI │ │ └─── Traditional AI predicts (classification, ranking, labeled data). Generative AI creates (text/images/code, FM-based, prompt-driven). │ └─── A4. Generative AI Business Value │ ├─── Creativity — Generating genuinely new content │ ├─── Productivity — Automating existing workflows │ └─── Connectivity — Improving user engagement ├─── B. Responsible AI Challenges │ ├─── B1. The Accuracy Problem: Bias and Variance │ │ └─── Bias = missing key features, causes underfitting. Variance = sensitivity to noise, causes overfitting. │ ├─── B2. The Bias-Variance Trade-off │ │ ├─── Underfitted — High Bias, Low Variance — the model is too simple to capture real patterns │ │ ├─── Overfitted — Low Bias, High Variance — the model has memorized noise instead of learning patterns │ │ └─── Balanced — Low Bias, Low Variance — the ideal state, generalizes well to new data │ ├─── B3. Mitigation Techniques │ │ ├─── Cross-Validation — Detects overfitting before deployment │ │ ├─── Increase Data — Improves learning by giving the model more to generalize from │ │ ├─── Regularization — Penalizes model complexity directly │ │ ├─── Simpler Models — Reduces overfitting by lowering capacity to memorize │ │ ├─── Dimensionality Reduction — Reduces the number of features the model must handle │ │ └─── Early Stopping — Halts training before the model starts memorizing noise │ └─── B4. Generative AI Challenges │ ├─── Toxicity — Generation of harmful content │ ├─── Hallucinations — False but plausible outputs │ ├─── Intellectual Property — Risk of duplicating protected content │ ├─── Plagiarism/Cheating — Misuse in academic or professional work │ └─── Work Disruption — Job displacement concerns ├─── C. Core Dimensions of Responsible AI │ ├─── C1. The Eight Dimensions │ │ ├─── Fairness — Avoiding discrimination in outcomes │ │ ├─── Explainability — Making WHY a decision was made understandable │ │ ├─── Transparency — Making HOW a decision was made visible │ │ ├─── Privacy & Security — Protecting the data a system touches │ │ ├─── Veracity & Robustness — Staying reliable under uncertainty │ │ ├─── Governance — Enforcing policy consistently │ │ ├─── Safety — Preventing harm to users or third parties │ │ └─── Controllability — Allowing behavior to be guided or corrected │ └─── C2. Business Benefits │ ├─── Trust & Reputation — User confidence in the system │ ├─── Regulatory Compliance — Meeting external legal requirements │ ├─── Risk Reduction — Fewer costly failures │ ├─── Competitive Advantage — Ethical practice as a market differentiator │ ├─── Better Decisions — More reliable outputs to act on │ └─── Improved Products & Business — Driving creative and innovative solutions ├─── D. AWS Tools for Responsible AI │ ├─── D1. SageMaker AI vs. Amazon Bedrock │ │ └─── Bedrock = generative AI platform. SageMaker = full ML lifecycle platform. The module's core distinction. │ ├─── D2. Foundation Model Evaluation │ │ ├─── Automatic Evaluation — Metrics: accuracy, robustness, toxicity │ │ ├─── Human Evaluation — Metrics: style, friendliness, brand alignment — via internal team or AWS reviewers │ │ └─── SageMaker AI Clarify — Supports automated FM evaluation, focused on responsible AI metrics and bias awareness │ ├─── D3. Guardrails for Amazon Bedrock │ │ ├─── Block Undesirable Topics — Restricts content outside an approved domain │ │ ├─── Filter Harmful Content — Blocks hate, violence, and abuse │ │ ├─── Redact PII — Protects personal data from exposure │ │ └─── Policy Enforcement — Applies custom rules per use case │ ├─── D4. Bias Detection and Data Balancing │ │ ├─── Random Undersampling — Reduces the majority class to match the minority │ │ ├─── Random Oversampling — Duplicates minority-class examples to match the majority │ │ └─── SMOTE — Synthetic data generation for underrepresented classes │ ├─── D5. Model Prediction Explanation │ │ └─── Explainability identifies key drivers of predictions — the module's key insight for this subsection │ ├─── D6. Monitoring and Human Review │ │ └─── Monitoring + human review = continuous Responsible AI — the module's key insight for this subsection │ ├─── D7. Governance Improvement Tools │ │ ├─── SageMaker Role Manager — Defines permissions across the ML lifecycle │ │ ├─── SageMaker Model Cards — Documents model details for governance review │ │ └─── SageMaker Model Dashboard — Monitors model behavior centrally │ └─── D8. Transparency Tools │ └─── Transparency = documentation + visibility — the module's key insight for this subsection ├─── E. Responsible Model Selection │ ├─── E1. Define the Use Case Narrowly │ │ └─── Use Case ≠ Technology — a use case must specify audience, output, and constraints │ ├─── E2. Performance Factors │ │ ├─── Customization — How well the model supports tuning │ │ ├─── Model Size — Parameter count, trading off capability and cost │ │ ├─── Latency — Response time under real load │ │ ├─── Context Window — Maximum usable input size │ │ └─── Licensing — Usage restrictions on the model │ ├─── E3. Dataset Consideration and Sustainability │ │ ├─── Energy Consumption — Direct carbon impact of running the model │ │ ├─── Resource Usage — Hardware demand required for operation │ │ └─── Environmental Impact — Long-term effects of sustained deployment │ ├─── E4. Responsible Agency │ │ ├─── Value Alignment — Consistency with human ethics │ │ ├─── Reasoning Skills — Capacity for moral decision-making │ │ ├─── Autonomy Level — Degree of human oversight retained │ │ └─── Accountability — Whether decisions remain transparent and traceable │ └─── E5. Economic Considerations │ ├─── Benefits — Efficiency gains from adoption │ ├─── Risks — Job displacement concerns │ └─── Market Impact — Broader inequality concerns from AI adoption at scale ├─── F. Responsible Dataset Preparation │ ├─── F1. Balanced Datasets and Inclusive Collection │ │ └─── Data bias = model bias — the module's core warning for this section │ ├─── F2. Data Curation │ │ ├─── Preprocessing — Cleaning data before it's usable │ │ ├─── Augmentation — Adding samples to strengthen underrepresented cases │ │ └─── Auditing — Continuous checks, not a one-time pass │ └─── F3. Use Case Alignment │ └─── Dataset must match the problem context — balance alone doesn't guarantee fit ├─── G. Transparent and Explainable AI │ ├─── G1.
