Three Fit States │ ├─── Overfitting — Performs well on training data, poorly on unseen data — caused by memorizing noise │ ├─── Underfitting — Performs poorly even on training data — caused by a model that's too simple │ └─── Balanced Model — Performs well on both training and unseen data — the goal: optimal generalization ├─── L. The Bias-Variance Trade-Off │ └─── L1. Bias vs. Variance │ ├─── High Bias → underfitting. High Variance → overfitting. │ └─── Bullseye analogy: center = true value, spread = variance, distance from center = bias ├─── M. Classification Metrics │ ├─── M1. The Confusion Matrix │ │ ├─── True Positive (TP) — A correct positive prediction │ │ ├─── True Negative (TN) — A correct negative prediction │ │ ├─── False Positive (FP) — An incorrect positive prediction │ │ └─── False Negative (FN) — A missed positive case │ ├─── M2. Accuracy, Precision, and Recall │ │ ├─── Accuracy — (TP+TN)/(TP+TN+FP+FN) — misleading with imbalanced data │ │ ├─── Precision — TP/(TP+FP) — use when false positives are costly, e.g. spam filtering │ │ └─── Recall (Sensitivity) — TP/(TP+FN) — use when false negatives are critical, e.g. disease detection │ └─── M3. F1 Score and AUC-ROC │ └─── AUC-ROC is threshold-independent — it shows performance across thresholds, not at one fixed cutoff ├─── N. Regression Metrics │ └─── N1. MSE and R² │ └─── MSE = error magnitude (lower is better). R² = goodness of fit (closer to 1 is better). They complement each other. ├─── O. Business Metrics and Model Evaluation │ ├─── O1.
